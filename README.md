## Обновление 1.2

**Исправлено:**
- Баг «бесконечное восстановление после перезагрузки»: если сервер был
  перезапущен (а вся игра живёт только в памяти процесса — токены пропадают),
  клиент раньше просто менял текст статуса и оставлял старое, уже мёртвое
  игровое поле висеть на экране без обновлений — выглядело как зависание или
  лаги. Теперь при неудачном восстановлении сессии страница просто чисто
  перезагружается и вы заходите заново.

**Переработано:**
- Учёный: раньше замораживал вражескую карту вместо атаки. Теперь при выходе
  замораживает **свою** карту (по умолчанию — самого себя, если не выбрать
  другую) — карта на 1 ход выключается, зато получает постоянные +1 HP/+1 ATK
  после разморозки (это тот же бонус, что и обычная заморозка). Стал
  саппорт-картой, а не помехой сопернику.
- «Ускорение» полностью переработано в «Общее ослабление»: раньше раз в
  5 раундов все теряли 5 HP, теперь **−2 HP всем участникам после КАЖДОГО
  хода** (не раз в 5 раундов, а буквально каждый раз, когда кто-то походил).
  Партии стали заметно короче и динамичнее — особенно с большим числом игроков,
  где ходов за раунд больше.
- Колода теперь одна на всю игру: раньше, когда колода заканчивалась, сброс
  перемешивался обратно. Теперь — нет: карты кончились, значит кончились,
  добор просто перестаёт что-либо давать (в лог пишется отдельное сообщение).
- Пропуск фазы действия («Пропустить действие», в том числе автоматический
  по таймеру) теперь даёт **+2 карты** в награду, а не 0.

**Новое:**
- 😀 Эмоции (как в Clash Royale): панель из 7 реакций (👍😂😮😡🤔🎉GG) над
  игровым полем — жмёте, все видят всплывающее сообщение «Имя: эмодзи».
  Не влияет на игру, чисто социальное.

**Обсуждается, не реализовано:**
- Интеграция с Supabase для аккаунтов — см. отдельный раздел ниже, это
  большая архитектурная работа, к которой стоит подойти отдельным заходом.

## Обновление 1.1

**Новые карты:** Снайпер (белая, игнорирует приоритеты/неуловимость), Рикошетер
(зелёная, доп. урон по соседней карте), Учёный (синяя — в 1.2 переработан,
см. выше), Шутка! (фиолетовая, неуязвимость+оглушение на игрока на 1 ход).

**Исправлено:**
- Оглушение (и заморозка) реально блокирует карту на её ход — раньше эффект
  снимался слишком рано (в начале хода, до фазы атаки) и фактически никогда
  никого не останавливал.
- Судья переработан: раньше была невнятная механика «раз за игру верните
  фиолетовую карту». Теперь при выходе он просто выдаёт в руку карту «Орест» —
  фиолетовую карту, которая при розыгрыше на 1 ход выводит на стол копию любой
  вашей белой/зелёной/синей карты из вашего сброса, после чего копия исчезает.

**Удобство игрока:**
- Отмена выбора: любая цель для способности/атаки теперь идёт через
  подтверждение («Играть эту карту / Атаковать» + «✕ Отмена»), плюс кнопка
  «Отмена» есть в каждой модалке выбора цели — если промахнулись по карте,
  можно откатиться и выбрать заново, ничего не отправляя на сервер раньше времени.
- 📖 Книга подсказок — кнопка сверху открывает список всех карт с их текстом
  способностей (тянется с сервера, `/cards-info.json`).
- Переподключение: при разрыве связи сервер держит место игрока (сообщает
  об этом остальным в логе), браузер сам пытается переподключиться и
  восстанавливает вас в той же партии по токену в localStorage — обновлять
  страницу и пересоздавать лобби не нужно.
- Кнопка «🚪 Выйти из игры» — осознанный выход, который освобождает место
  (в лобби — просто удаляет вас; в разгаре партии — засчитывает выбывание).
- ⏳ Таймер хода: на фазы действия и атаки даётся 60 секунд (видно отсчёт в
  шапке). Не успел — фаза автоматически пропускается (как обычный «Пропустить
  действие» / «Не атаковать»), и ход идёт дальше. Поменять лимит — константа
  `TURN_SECONDS` в `engine.js`.

**Баланс:** Дан — Приоритет 3→2, HP 1→3. Солдат Данил — бонус за выживание
хода +4→+2.

**Отложено по вашей просьбе:** полноценные множественные лобби (создание/поиск/
имя лобби) — сейчас на сервере одна общая партия на всех подключившихся.

## Запуск

1. Установить Node.js (если ещё нет): https://nodejs.org
2. В папке проекта выполнить:

   ```bash
   npm install
   npm start
   ```

3. Открыть в браузере: http://localhost:8080
4. Открыть эту же страницу в нескольких вкладках/устройствах в той же сети,
   ввести имя и нажать «Присоединиться» в каждой.
5. Когда все готовы (минимум 2 игрока) — любой игрок нажимает «Начать игру».
6. Каждый выбирает одного из двух предложенных боссов.
7. Играем по фазам: добор — автоматически, действие и атака — по кнопкам/картам.

## Как устроено

- `cards.js` — база всех карт: характеристики и текст способностей.
- `engine.js` — весь игровой движок: очередь ходов, фазы, приоритеты атаки,
  статусы (яд, вирус, оглушение, заморозка, неуязвимость, стена и т.д.),
  проверка побед/выбываний, реконнект по токену, авторитетное состояние игры.
- `server.js` — WebSocket-обвязка: принимает сообщения от клиентов, вызывает
  методы `engine.js`, рассылает каждому игроку персональное состояние (руки
  других игроков скрыты, видно только количество карт), отдаёт `/cards-info.json`.
- `index.html` — интерфейс: стол всех игроков, рука, модалки выбора целей
  для способностей, книга подсказок, подтверждение/отмена действий.

## Что реализовано

Все белые, зелёные, синие, жёлтые (боссы) и фиолетовые карты, включая карты
1.1/1.2 (коричневые тестовые карты по правилам не участвуют). Фазы хода,
приоритеты целей (Даны → боссы → «Приоритет 1» → остальные), неуловимость,
меткость (Снайпер), яд, вирус, оглушение, заморозка + бонус после неё,
неуязвимость (карт и игроков), Стена (50%), Рост Обычного Данила, «Общее
ослабление» −2 HP всем после каждого хода (1.2), выбывание игроков и возврат
их карт в колоды, комбо «карта + фиолетовая», конечная (без пересдачи) колода.

## Известные допущения (правила не давали точных цифр/порядка)

- **Количество копий карт в колоде**: не указано в правилах, взято по 3 копии
  каждой белой/зелёной/синей карты, по 2 — каждой фиолетовой (кроме «Ореста» —
  он не тянется из колоды, только от Судьи), по 3 копии каждого босса.
  Легко поменять в `cards.js` (`DECK_COUNTS`, `BOSS_COPIES`). С 1.2 колода не
  пересдаётся из сброса, так что при большом числе игроков карты реально
  могут закончиться до конца партии — это теперь штатное поведение, не баг.
- **«Орест»** не изымает исходную карту из сброса — просто временно копирует
  её характеристики на 1 ход (иначе пришлось бы вести двойной учёт сброса
  для переразмешивания колоды и для Ореста одновременно).
- **«Рикошетер»**: «соседняя карта» — следующая по порядку карта в массиве
  стола соперника (или предыдущая, если цель была последней в списке).
- **«Учёный»** (переработан в 1.2): если не выбрать конкретную свою карту,
  замораживает сам себя.
- **«Шутка!»**: оглушение игрока означает полный пропуск его следующего хода
  (добор/действие/атака), неуязвимость не блокирует «Общее ослабление» (оно
  прямым текстом не блокируется ничем по правилам).
- **Атака «самим собой»** (когда стол пуст) доступна каждый ход по выбору
  игрока, база урона 1, с теми же правилами приоритета целей.
- **Множественные соперники**: при атаке или эффекте, требующем «врага»,
  игрок сначала выбирает конкретного соперника (если их больше одного),
  затем — карту на его столе.
- Порядок применения защиты при уроне: разовый блок Рыбака → Стена (50%) →
  вычитание из HP; отражение (Данил с длинной палкой) считается от уже
  уменьшённого урона и не влияет на урон исходной атаки.

## Почему отложено

- **Полноценные множественные лобби** (создание/поиск/имя/выход из конкретного
  лобби) — сейчас на сервере одна общая игра на всех подключившихся
  (`const game = new Game()` в `server.js`). Чтобы поддержать несколько
  независимых партий одновременно, нужно превратить это в `Map<lobbyId, Game>`
  плюс экран со списком лобби на клиенте — существенный, но понятный по объёму
  кусок работы, который стоит делать отдельным заходом.

## Про Supabase (аккаунты)

Сейчас у игры вообще нет постоянного хранилища — вся партия живёт в
оперативной памяти процесса Node и полностью исчезает при перезапуске
сервера (см. фикс 1.2 выше — раньше это выглядело как баг, теперь хотя бы
корректно обрабатывается, но сама потеря данных никуда не делась).

Supabase решил бы это по-настоящему: логин/пароль или OAuth, профиль игрока,
история партий, возможно — сохранение и возобновление партии после
перезапуска сервера. Но это довольно большая по объёму работа:

- Аутентификация на клиенте (Supabase Auth) + защита WebSocket-сессий
- Схема БД: игроки, партии, статистика
- Сериализация всего состояния `Game`/`Player`/карт в БД и обратно
  (сейчас всё это — живые JS-объекты с циклическими ссылками на `ws`,
  просто так в JSON не положишь — нужен отдельный слой (де)сериализации)
- Решить, что вообще значит «аккаунт» в игре, где партия — это разовая
  комната на несколько человек, а не что-то персональное

Это отдельный проект по объёму сравнимый с мультилобби (может и больше).
Предлагаю обсудить отдельно, когда до него дойдёт очередь — есть смысл
сначала определиться, нужны ли аккаунты сами по себе (для статистики?
для чего-то ещё?), или вам просто нужно, чтобы партия переживала перезапуск
сервера — тогда это решается проще, без полноценных аккаунтов (просто
сохранение состояния игры в файл/БД и его восстановление при старте).
