# Мир Данилов 2 — мультиплеер

## Обновление 1.1

**Новые карты:** Снайпер (белая, игнорирует приоритеты/неуловимость), Рикошетер
(зелёная, доп. урон по соседней карте), Учёный (синяя, вместо атаки замораживает
врага на 1 ход), Шутка! (фиолетовая, неуязвимость+оглушение на игрока на 1 ход).

**Исправлено:**
- Оглушение (и заморозка) реально блокирует карту на её ход — раньше эффект
  снимался слишком рано (в начале хода, до фазы атаки) и фактически никогда
  никого не останавливал.
- Судья переработан: раньше была невнятная механика «раз за игру верните
  фиолетовую карту». Теперь при выходе он просто выдаёт в руку карту «Орест» —
  фиолетовую карту, которая при розыгрыше на 1 ход выводит на стол копию любой
  вашей белой/зелёной/синей карты из вашего сброса, после чего копия исчезает.

**Удобство игрока:**
- Отмена выбора: любая цель для способности/атаки теперь идёт через
  подтверждение («Играть эту карту / Атаковать» + «✕ Отмена»), плюс кнопка
  «Отмена» есть в каждой модалке выбора цели — если промахнулись по карте,
  можно откатиться и выбрать заново, ничего не отправляя на сервер раньше времени.
- 📖 Книга подсказок — кнопка сверху открывает список всех карт с их текстом
  способностей (тянется с сервера, `/cards-info.json`).
- Переподключение: при разрыве связи сервер держит место игрока (сообщает
  об этом остальным в логе), браузер сам пытается переподключиться и
  восстанавливает вас в той же партии по токену в localStorage — обновлять
  страницу и пересоздавать лобби не нужно.
- Кнопка «🚪 Выйти из игры» — осознанный выход, который освобождает место
  (в лобби — просто удаляет вас; в разгаре партии — засчитывает выбывание).

**Баланс:** Дан — Приоритет 3→2, HP 1→3. Солдат Данил — бонус за выживание
хода +4→+2.

**Отложено (см. пояснение ниже):** таймер на ход, полноценные множественные
лобби (создание/поиск/имя лобби).

## Запуск

1. Установить Node.js (если ещё нет): https://nodejs.org
2. В папке проекта выполнить:

   ```bash
   npm install
   npm start
   ```

3. Открыть в браузере: http://localhost:8080
4. Открыть эту же страницу в нескольких вкладках/устройствах в той же сети,
   ввести имя и нажать «Присоединиться» в каждой.
5. Когда все готовы (минимум 2 игрока) — любой игрок нажимает «Начать игру».
6. Каждый выбирает одного из двух предложенных боссов.
7. Играем по фазам: добор — автоматически, действие и атака — по кнопкам/картам.

## Как устроено

- `cards.js` — база всех карт: характеристики и текст способностей.
- `engine.js` — весь игровой движок: очередь ходов, фазы, приоритеты атаки,
  статусы (яд, вирус, оглушение, заморозка, неуязвимость, стена и т.д.),
  проверка побед/выбываний, реконнект по токену, авторитетное состояние игры.
- `server.js` — WebSocket-обвязка: принимает сообщения от клиентов, вызывает
  методы `engine.js`, рассылает каждому игроку персональное состояние (руки
  других игроков скрыты, видно только количество карт), отдаёт `/cards-info.json`.
- `index.html` — интерфейс: стол всех игроков, рука, модалки выбора целей
  для способностей, книга подсказок, подтверждение/отмена действий.

## Что реализовано

Все белые, зелёные, синие, жёлтые (боссы) и фиолетовые карты, включая карты
1.1 (коричневые тестовые карты по правилам не участвуют). Фазы хода, приоритеты
целей (Даны → боссы → «Приоритет 1» → остальные), неуловимость, меткость
(Снайпер), яд, вирус, оглушение, заморозка + бонус после неё, неуязвимость
(карт и игроков), Стена (50%), Рост Обычного Данила, «Ускорение» каждые
5 раундов (−5 HP всем), выбывание игроков и возврат их карт в колоды, комбо
«карта + фиолетовая».

## Известные допущения (правила не давали точных цифр/порядка)

- **Количество копий карт в колоде**: не указано в правилах, взято по 3 копии
  каждой белой/зелёной/синей карты, по 2 — каждой фиолетовой (кроме «Ореста» —
  он не тянется из колоды, только от Судьи), по 3 копии каждого босса.
  Легко поменять в `cards.js` (`DECK_COUNTS`, `BOSS_COPIES`).
- **«Орест»** не изымает исходную карту из сброса — просто временно копирует
  её характеристики на 1 ход (иначе пришлось бы вести двойной учёт сброса
  для переразмешивания колоды и для Ореста одновременно).
- **«Рикошетер»**: «соседняя карта» — следующая по порядку карта в массиве
  стола соперника (или предыдущая, если цель была последней в списке).
- **«Учёный»**: заморозка вместо атаки не подчиняется приоритету целей (это
  отдельная способность, а не атака).
- **«Шутка!»**: оглушение игрока означает полный пропуск его следующего хода
  (добор/действие/атака), неуязвимость не блокирует «Ускорение» (оно прямым
  текстом не блокируется ничем по правилам).
- **Атака «самим собой»** (когда стол пуст) доступна каждый ход по выбору
  игрока, база урона 1, с теми же правилами приоритета целей.
- **Множественные соперники**: при атаке или эффекте, требующем «врага»,
  игрок сначала выбирает конкретного соперника (если их больше одного),
  затем — карту на его столе.
- Порядок применения защиты при уроне: разовый блок Рыбака → Стена (50%) →
  вычитание из HP; отражение (Данил с длинной палкой) считается от уже
  уменьшённого урона и не влияет на урон исходной атаки.

## Почему отложено

- **Таймер на ход** — технически несложно (таймаут на сервере), но нужно
  сначала решить игровой вопрос: что происходит при просрочке — авто-пропуск,
  авто-атака первой картой, или что-то ещё? Как определитесь — добавлю быстро.
- **Полноценные множественные лобби** (создание/поиск/имя/выход из конкретного
  лобби) — сейчас на сервере одна общая игра на всех подключившихся
  (`const game = new Game()` в `server.js`). Чтобы поддержать несколько
  независимых партий одновременно, нужно превратить это в `Map<lobbyId, Game>`
  плюс экран со списком лобби на клиенте — существенный, но понятный по объёму
  кусок работы, который стоит делать отдельным заходом.

## Что можно добавить дальше
